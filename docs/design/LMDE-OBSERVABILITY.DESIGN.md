## Overview

The LMDE Observability Stack provides a local, permanent, and secure destination for metrics and traces generated by coding agents (e.g., `clai`) and other local services. It utilizes a `kind` cluster for isolation, a local container registry for supply-chain resistance, and industry-standard tools (OTel, Prometheus, Grafana) for the data pipeline.

---

## Goals

1. **Persistent Metrics** -- Metrics survive `kind` restarts, host reboots, and cluster recreations via host-path mapping.
2. **Supply-Chain Resistance** -- All images are pinned by SHA256 digest and served from a local registry (`localhost:5001`).
3. **Unified Ingress** -- A single OTel collector endpoint (`localhost:4317`/`4318`) for all local telemetry.
4. **Blast Radius Control** -- Resource quotas and limits ensure observability services do not exhaust host resources.
5. **Zero-Configuration for Consumers** -- Any tool on the laptop can assume the OTLP endpoint is available.

---

## Non-Goals

- **Log Aggregation** -- This stack focuses on metrics and traces. Log management is handled by `log-hoarder`.
- **Multi-node Scalability** -- Designed for single-user laptop scale.
- **Public Ingress** -- The stack is strictly local and not exposed to the internet.

## Design

### 1. Local Container Registry

Ensures that the environment is hermetic and resistant to upstream poisoning or outages.

#### Responsibilities

| Responsibility | Details |
|----------------|---------|
| Image Mirroring | Pulls upstream images, verifies digests, and pushes to `localhost:5001`. |
| Digest Pinning | All images are referenced by `name@sha256:hash` in `images.txt`. |
| Lifecycle | Started as a standalone Docker container before `kind` to avoid circularity. |

### 2. Infrastructure (kind)

Provides the isolation boundary and resource management.

#### Configuration (`kind-config.yaml`)
- **Nodes**: 1 control-plane node.
- **Port Mappings**:
  - `3000` -> `3000` (Grafana UI)
  - `4317` -> `4317` (OTLP gRPC)
  - `4318` -> `4318` (OTLP HTTP)
- **Mounts**:
  - `~/.local/share/tds-utils/observability/data` -> `/mnt/data` (Persistence)

### 3. Telemetry Pipeline (OTel + Prometheus + Grafana)

The core data processing and visualization stack.

#### Deployment Strategy
- **Helm**: Use `prometheus-community` and `grafana` charts, but template them to use local registry images.
- **Persistence**: Configure charts to use a `HostPath` PersistentVolume pointing to `/mnt/data`.
- **Quotas**: A `ResourceQuota` in the `observability` namespace limits CPU to 1.0 and Memory to 2Gi.

## State Machine: Bootstrap Flow

```
+----------+      +------------+      +------------+      +-----------+
| Registry |----->| Kind Cluster|----->| Helm Charts|----->| Readiness |
+----------+      +------------+      +------------+      +-----------+
```

| From | To | Trigger | Condition |
|------|-----|---------|-----------|
| START | Registry | `setup.sh` | Docker is running |
| Registry | Kind Cluster | Registry is healthy | `images.txt` is synced |
| Kind Cluster | Helm Charts | Cluster is Ready | HostPath dir exists |
| Helm Charts | Readiness | Pods are Running | Smoke tests pass |

## Security Considerations

- **Supply Chain** -- Mitigated by local registry + digest pinning.
- **Resource Exhaustion** -- Mitigated by K8s ResourceQuotas.
- **Data Privacy** -- All data stays on the local disk; no "phone home" analytics in Grafana/Prometheus.

---

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Storage | HostPath Mapping | Simplest path to survival across cluster deletes in `kind`. |
| Ingress | kind extraPortMappings | Avoids complexity of an Ingress Controller for a local stack. |
| Registry | Registry:2 (standalone) | Avoids circular dependency where the cluster needs the registry to start the registry. |
| Deployment | Helm (Local Images) | Leverages community best-practices for config while maintaining image control. |

## Rejections

- **Dynamic Provisioning (EBS/GCP)** -- Not applicable to a local laptop environment.
- **Ingress-Nginx** -- Overkill for 3 static ports on localhost.
- **Local Registry as a Pod** -- Circular dependency risk.
